# 컨테이너 모니터링 시각화 대시보드
## Docker & Kubernetes 환경 실시간 모니터링 솔루션

Docker와 Kubernetes 환경에서 컨테이너와 인프라의 상태를 실시간으로 모니터링하고 시각화하는 웹 기반 통합 대시보드입니다.

## 🎯 프로젝트 개요

컨테이너 오케스트레이션 환경의 복잡성이 증가함에 따라 시스템 운영자가 전체 인프라 상황을 한눈에 파악하고 신속한 의사결정을 할 수 있는 직관적인 모니터링 솔루션의 필요성이 대두되었습니다. 본 프로젝트는 이러한 요구사항을 해결하기 위한 실시간 통합 모니터링 대시보드입니다.

## 📊 주요 기능 (웹 화면 순서)

### 1️⃣ 메인 대시보드 (Overview)
**핵심 지표 실시간 모니터링**
- 전체 컨테이너: 148개 (+12 증가)
- 실행 중 컨테이너: 135개 (91.2% 가동률)
- 중지된 컨테이너: 6개 / 실패한 컨테이너: 5개
- 활성 노드: 12개 클러스터
- 평균 CPU 사용률: 32.5% / 평균 메모리: 68.2%
- 네트워크 트래픽: 142 MB/s

**실시간 차트**
- CPU/메모리 사용률 추이 (라인 차트)
- 네트워크 트래픽 모니터링 (최대 200MB/s)
- 노드별 컨테이너 분포 (도넛 차트)

### 2️⃣ 컨테이너 관리 (Containers)
- 전체 컨테이너 목록 및 상세 정보
- 컨테이너별 CPU/메모리 사용량
- 실시간 상태 업데이트 (Running/Stopped)
- 컨테이너 제어 액션 (시작/중지/상세보기)

### 3️⃣ 노드 관리 (Nodes)
- 클러스터 노드 현황: 정상 11개(91.7%), 주의 1개(8.3%)
- 노드별 리소스 사용률 (CPU/메모리/디스크)
- 리소스 사용률 히트맵
- SSH 접속 및 노드 관리 기능

### 4️⃣ 고급 모니터링 (Advanced Monitoring)
- 디스크 I/O 사용률 추이
- 응답 시간 분석 차트
- 요청 상태 분포 (성공/실패)
- Top 리소스 사용 컨테이너 랭킹

### 5️⃣ 로그 관리 (Logs)
- 실시간 통합 로그 스트림
- 로그 레벨별 색상 구분 (INFO/WARN/ERROR)
- 컨테이너별 로그 필터링

### 6️⃣ 알림 관리 (Alerts)
- 알림 현황: Critical 2개, Warning 5개, Info 12개
- 활성 알림 목록 및 관리
- 알림 규칙 설정 (CPU/메모리/디스크 임계값)
- 다채널 알림 지원 (이메일/Slack/SMS/웹훅)

### 7️⃣ 이벤트 추적 (Events)
- 오늘 이벤트: 47개 (+12 vs 어제)
- 실시간 이벤트 타임라인
- 이벤트 타입별 필터링 및 검색
- 이벤트 발생 추이 분석

## 🎨 디자인 특징

- **카드 호버 효과**: 인터랙티브한 사용자 경험
- **실시간 표시기**: 펄스 애니메이션으로 실시간 상태 표시
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 지원
- **부드러운 애니메이션**: CSS 트랜지션 효과

## 🚀 실행 방법

### 1. 브라우저에서 직접 열기
```bash
# dashboard 폴더의 index.html 파일을 브라우저에서 열기
open dashboard/index.html
```

### 2. 로컬 서버로 실행 (권장)
```bash
# Python 내장 서버 사용
cd dashboard
python -m http.server 8000

# 또는 Node.js http-server 사용
npx http-server -p 8000
```

### 3. 접속
브라우저에서 `http://localhost:8000` 접속

## 📱 반응형 지원

- **태블릿**: 768px-1199px - 2열 레이아웃
- **모바일**: 767px 이하 - 1열 레이아웃

## 🔧 기술 스택

- **HTML5**: 시맨틱 마크업
- **CSS3**: Flexbox, Grid, 애니메이션
- **JavaScript**: Chart.js 차트 라이브러리
- **Bootstrap 5**: 반응형 UI 프레임워크
- **Font Awesome**: 아이콘 라이브러리

## 📈 차트 라이브러리

**Chart.js**를 사용하여 다음 차트 타입 구현:
- **Line Chart**: 시계열 데이터 (CPU/메모리/네트워크)
- **Bar Chart**: 카테고리별 비교 (컨테이너별 사용률)
- **Doughnut Chart**: 비율 표시 (노드별 분포)
- **Pie Chart**: 비율 표시 (엔진별 분포)
